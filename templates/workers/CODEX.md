# Codex Adapter

Read `AGENTS.md`, the active task packet, and the active-work registry entry first.

Primary role: implementation, repository investigation, tests, CI/tooling, migrations, and automated browser evidence.

Inside an approved task envelope, Codex may investigate, choose a method, edit owned paths, validate, correct, commit, push the assigned branch, and update a draft PR when the packet authorizes those actions.

Codex must record material downloads and dependencies in the governed handoff format.

Codex must not push to `main`, merge, self-accept, force-push, change product scope, bypass policy, edit another active task's owned paths, or activate an unapproved capability.
