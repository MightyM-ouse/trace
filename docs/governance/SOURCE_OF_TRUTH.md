# Source Of Truth

TRACE treats the repository as the canonical collaboration record.

## Precedence

1. Product Owner or maintainer decisions recorded in repository artifacts.
2. Accepted requirements, architecture, specifications, and security policy.
3. Approved task packet plus recorded amendments.
4. Current code, tests, configuration, and migrations.
5. Current-state map, active-work registry, and curated evidence.
6. README files and historical reports.

Chat sessions, local scratch folders, generated archives, and model memory are not authoritative project state.

## Current-State Claims

Before answering about current project state, inspect the applicable branch, issue or PR, `AGENTS.md`, `trace/STAR_MAP.md`, and routed sources. If access is unavailable, state the limitation and avoid presenting cached context as current.

An implemented behavior does not automatically supersede an accepted requirement. A documented requirement does not prove implementation. Report conflicts explicitly.
